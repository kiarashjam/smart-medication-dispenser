Why ESP32-S3?

  • ✓
    Integrated WiFi + BLE

    Reduces BOM, no external RF module needed

  • ✓
    Dual-core 240MHz

    Handles UI + networking simultaneously

  • ✓
    8MB PSRAM

    Large buffers for audio, display graphics

  • ✓
    Hardware Crypto

    AES, SHA, RSA acceleration for security

  • ✓
    USB OTG

    Direct firmware programming, diagnostics

  • ✓
    10µA Deep Sleep

    Essential for battery backup mode

  • ✓
    OTA Support

    Built-in dual-partition firmware update

Pure Standby Mode

5000mAh ÷ 20mA = 250 hours ≈ 10 days

With 4 Dispenses/Day (realistic use)

Daily consumption: Standby: 20mA × 23.9h = 478mAh Dispense: 940mA × 0.1h = 94mAh Total: 572mAh/day 5000mAh ÷ 572mAh = ~48 hours ✓

Common Pitfalls & Solutions

Learn from our experience

Motor Noise

Problem: Stepper causes EMI, affects WiFi

Solution: Add 100nF across each motor coil, ferrite beads on motor power

WiFi Interference

Problem: Motors affect WiFi performance

Solution: Separate power paths, star ground, proper filtering

Power Sequencing

Problem: Brown-out when motor starts

Solution: Add inrush current limiting, bulk capacitors on motor rail

Touch Sensitivity

Problem: Erratic touch input

Solution: Ground display bezel, use shielded FFC cable

Battery Drain

Problem: Won't last 48h backup

Solution: Proper sleep modes, kill power to peripherals with load switches

Load Cell Drift

Problem: Weight readings vary with temperature

Solution: Temperature compensation, periodic tare calibration
