## 1. Regulatory Strategy Overview

### 1.1 Why CE First?

| Approach | Rationale |
|:---------|:----------|
| CE marking primary | Access to 450M+ population market |
| Swissmedic via MRA | Simplified path after CE |
| No FDA initially | European focus reduces complexity/cost |
| ISO 13485 foundation | Required for all pathways |

### 1.2 Device Classification

| Jurisdiction | Classification | Basis |
|:-------------|:---------------|:------|
| **EU MDR** | Class IIa | Active device for medication dispensing, continuous monitoring |
| **Swissmedic** | Class IIa | Aligned with EU classification |
| **Comparable devices** | Class IIa | Medido, Philips dispenser, Hero |

Classification rationale under EU MDR 2017/745:
- Rule 11: Active device intended for diagnosis, monitoring, or treatment
- Rule 12: Active device administering medicinal products
- Not implantable, not Class III (no critical diagnosis)

## 7. Swissmedic Registration

### 7.1 Swiss Medical Device Ordinance (MedDO 2020)

Since January 2021, Switzerland has updated its medical device regulation to align with EU MDR.

| Requirement | Details |
|:------------|:--------|
| **CE marking required** | Yes (with Notified Body) |
| **Swiss CH-REP** | Required if non-Swiss manufacturer |
| **Registration** | Via Swissmedic online portal |
| **Fees** | CHF 2,000-8,000 depending on class |
| **Timeline** | 30-60 days after CE |

### 7.2 Swissmedic Registration Process

| Step | Duration | Details |
|:-----|:---------|:--------|
| 1. Obtain CE marking | — | Prerequisite |
| 2. Appoint CH-REP (if needed) | 1 week | Swiss Authorized Representative |
| 3. Prepare registration dossier | 1-2 weeks | Per MedDO requirements |
| 4. Submit to Swissmedic | 1 day | Online portal |
| 5. Swissmedic review | 30-60 days | May request clarification |
| 6. Registration confirmed | — | Device may be marketed |

### 7.3 Swissmedic Costs

| Item | Cost |
|:-----|-----:|
| CH-REP services (annual) | CHF 5,000 |
| Registration fee | CHF 3,000 |
| Documentation preparation | CHF 5,000 |
| Annual maintenance | CHF 2,000/year |
| **Total (initial)** | **CHF 13,000** |

## 8. Data Protection & Cybersecurity

### 8.1 GDPR Compliance

| Requirement | Implementation |
|:------------|:---------------|
| **Legal basis** | Consent + legitimate interest |
| **Data minimization** | Collect only necessary data |
| **Purpose limitation** | Clear, documented purposes |
| **Storage limitation** | 7-year retention, then deletion |
| **Security** | Encryption, access controls |
| **Data subject rights** | Export, deletion capability |
| **DPO** | Appointed when >10 employees |
| **Data processing agreements** | With all sub-processors |

### 8.2 Swiss nDSG Compliance

The new Swiss Data Protection Act (nDSG, effective Sept 2023) aligns with GDPR:

| Requirement | Implementation |
|:------------|:---------------|
| Transparency | Privacy policy in FR/DE/IT |
| Processing register | Maintained and updated |
| DPIAs | For high-risk processing |
| Cross-border transfers | Only to adequate countries |
| Breach notification | Within 72 hours to FDPIC |

### 8.3 EU MDR Cybersecurity Requirements

| Requirement | Our Approach |
|:------------|:-------------|
| Risk analysis | Threat modeling, STRIDE |
| Security controls | Defense in depth |
| Secure development | IEC 62443 principles |
| Update mechanism | Signed OTA updates |
| Vulnerability management | Monitoring, patching |
| Incident response | Documented procedures |

### 8.4 Cybersecurity Technical Controls

| Layer | Control | Implementation |
|:------|:--------|:---------------|
| **Device** | Secure boot | Verified boot chain |
| **Device** | Encrypted storage | AES-256 |
| **Communication** | TLS 1.3 | Certificate pinning |
| **Authentication** | Device tokens | JWT, rotation |
| **Cloud** | Access control | RBAC, MFA |

|:-----|:------------|:-------|:-----------|
| Notified Body backlog | Medium | High | Apply to multiple NBs |
| Test failures | Low | Medium | Pre-testing, design margin |
| MDR interpretation changes | Low | Medium | Track guidance updates |

### 12.2 Contingency Plans

| Scenario | Response |
|:---------|:---------|
| NB delay >2 months | Switch to faster NB |
| Test failure | Redesign, re-test with buffer |
| Budget overrun | Reduce scope, prioritize CE |
